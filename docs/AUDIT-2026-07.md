# cenno full audit ‚Äî July 2026

Scope: architecture, security (Rust backend + frontend/webview), privacy & data handling, supply chain & release. Method: four parallel independent review passes over the full codebase (`src-tauri/`, `src/`, `companion/`, config, release pipeline), findings cross-checked against SECURITY.md's threat model, then the whole report verified with Codex (corrections incorporated below).

## Verdict

The core is in good shape. The agent boundary ‚Äî the part the threat model centers on ‚Äî is soundly built and well tested:

- **No exploitable vulnerabilities found** in the Rust backend or the webview render path (shipped builds).
- SQL is fully parameterized; no command injection or path traversal sinks reachable by agent input; Tauri capabilities are minimal and agent traffic never reaches `#[tauri::command]`s (agents only get the 0600 Unix socket).
- A2UI payloads are rejected at the MCP boundary (`a2ui_guard.rs`, 17 tests), rendered as declarative React props (no `dangerouslySetInnerHTML`, react-markdown without `rehype-raw`, default URL transform strips `javascript:`/`file:` links), backstopped by a restrictive CSP in bundled builds.
- Updater flow is sound: user-initiated only, pinned minisign pubkey, HTTPS GitHub endpoint, signature verified before install.
- Production Rust is nearly free of `unwrap`/`expect` (a handful remain in CLI/startup paths ‚Äî `main.rs:72`, `main.rs:82`, `mcp.rs:29` ‚Äî all deterministic/low-risk); the decision logic (`suppress.rs`, `routing.rs`, `registry.rs`) is pure, injectable, and the best-tested code in the repo.

**The real findings are privacy-disclosure and completeness issues, not code vulnerabilities.** The CloudKit relay (companion feature) undermines the app's central privacy promise as currently shipped, and the relay itself is half-built.

## Key findings

### P0 ‚Äî Privacy promise vs. CloudKit relay

1. **Relay routing is ON by default and would ship full prompt+answer content to iCloud in an entitled build.** `RoutingConfig::default()` sets `iphone: Fallback` (`src-tauri/src/routing.rs:66,90`), so every `ask_user` serializes the entire `AskRequest` JSON and later the full answer into private-database CloudKit records ‚Äî no opt-in, no companion required. **Important scoping (verified):** the shipped Mac build cannot actually publish ‚Äî the iCloud/CloudKit entitlements were removed in 0.3.1 (`src-tauri/Entitlements.plist`; restricted entitlements SIGKILLed the Developer-ID 0.3.0 build) and `CennoRelay.swift:36` skips writes without the entitlement. So this is a latent dangerous default, not active exfiltration today ‚Äî but the entitlement-gated OFF state, default-ON routing config, compiled relay code, and shipping companion app form an **entitlement/config mismatch** that must be resolved deliberately before CloudKit is ever re-enabled. It also contradicts README ("never leaves the machine") and SECURITY.md ("no network connections on its own; the one exception is the update check") as written.
2. **README/SECURITY.md never mention the relay.** Zero references to CloudKit/companion/iCloud in the threat model or README privacy sections.
3. **CloudKit records are never deleted** ‚Äî neither side purges (`modifyRecords(... deleting: [])` on both Mac and companion); `expires_at` is a data field, not a TTL. When the relay is enabled, prompt/answer history accumulates in iCloud indefinitely with no erase.
4. **The relay payload is unredacted.** `payload_for_relay` (`mcp.rs:143`) ships the whole `AskRequest`, not the fields the companion renders ‚Äî payload minimization belongs in the same P0 fix, not a later cleanup.

### P1 ‚Äî Product/robustness

5. **The relay round-trip is incomplete on the Mac side.** The companion *does* submit answers into CloudKit (`companion/Sources/Shared/CloudKitRelay.swift:52`), but nothing on the Mac subscribes/polls to ingest them into `registry.resolve` ‚Äî the parked `ask_user` burns its full timeout regardless. Until the round-trip works, the relay is a one-way mirror.
6. **No local retention/erase.** No DELETE path in `db.rs`, no CLI or settings action to clear history; export exists, deletion doesn't. Note: a bare `DELETE` is insufficient for plaintext answers (`prompts.answer`, `db.rs:158`) ‚Äî deleted rows persist in SQLite free pages, so clearing must `VACUUM` (or equivalent compaction) afterward.
7. **`PromptRegistry` never evicts** consumed/expired entries (acknowledged in `registry.rs:15`); slow leak in a long-running menu-bar app.
8. **Client cancellation not observed** (`mcp.rs:133` TODO): a dead agent leaves the panel on screen for the full timeout.
9. **Single-instance race** (`mcp.rs:346` TODO): concurrent launches can unlink each other's live socket.
10. **`ask_sequence` never publishes to the relay** (`mcp.rs:270` vs `mcp.rs:156`) ‚Äî inconsistent with `ask_user`, silently invisible to companions.
11. **Relay writes are fire-and-forget** with a create/update race in `CennoRelay.swift` ‚Äî a lost update leaves the companion showing a stale pending prompt.

### P2 ‚Äî Supply chain, hygiene, hardening

12. **No CI.** `.github/workflows/` is empty; the test suite is enforced by nothing. A release risk, not just hygiene.
13. **TTS model download integrity is size-only** from a mutable HuggingFace `resolve/main` ref (`supertonic/mod.rs:294`); should pin a revision + SHA-256. HF and the Apple speech-model downloads are also undisclosed network endpoints in README's "one exception" claim.
14. **31 MB of release binaries committed** (`dist-release-0.3.0/`, incl. a stale `latest.json` saying 0.3.0 while the repo is at 0.3.2); `.gitignore` misses `dist-release-*`.
15. **Risky deps:** `tauri-nspanel` pinned to a git rev of an unpublished single-maintainer branch; `ort` at a pre-release RC.
16. **`ExternalLink` scheme allowlist** would be defense-in-depth on top of react-markdown's transform (`src/a2ui/views.tsx`).
17. **`dismiss_pending` is cross-agent** ‚Äî any connected client can dismiss all agents' prompts (accepted same-user trust boundary; note only).
18. **No corrupt-DB handling mid-run**; settings writes fail silently after corruption.
19. **Dead weight:** `spike/a2ui/` (68 KB, superseded, still cited by a live comment in `src/a2ui/desugar.ts`), tracked `companion/.enzyme/` artifacts.
20. **Untested glue:** `bridge.rs` connect-or-launch, `updater.rs`, `relay.rs`, companion `CloudKitRelay` have zero tests; `bridge.rs` and the relay are the testable, robustness-critical ones.

(Verified non-issue: the Homebrew cask referenced by AGENTS.md **does** exist ‚Äî `glebis/homebrew-tap` `Casks/cenno.rb`; an earlier draft finding claiming otherwise was dropped.)

### Dev-build-only notes (no action beyond docs)

- CSS injection via `~/.cenno/tokens.json` values (user's own config, CSP-contained in bundles).
- Agent-supplied `Image.url` loads remotely only in `tauri dev` (CSP blocks it in bundles).
- Socket perms set post-bind (sub-ms window, same-user boundary anyway).
