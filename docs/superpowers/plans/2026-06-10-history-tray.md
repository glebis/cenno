# Cenno History & Tray Implementation Plan (plan 4a — pulled forward by user goal)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal (user's words):** SQLite history table + export CLI; tray icon created using Codex, visible in the tray; tray menu with pause options (15 min / 30 min / 1 h / 2 h / 5 h / 8 h / until tomorrow); "don't show in fullscreen mode" on by default — if another app is fullscreen, cenno shows nothing (prompts queue quietly).

**Architecture:** rusqlite database at `{app_data_dir}/cenno.db` (prompts history + settings). Recording happens in the ask_user tool path after `registry.ask()` returns. `cenno export` reads the DB headless (no app needed), JSON/CSV to stdout. Tray via Tauri 2's `tray-icon` feature with a template (monochrome) menu-bar icon generated by Codex CLI. Pause and fullscreen-suppression gate the *display* (notify path) only — prompts still register and the cold-start replay machinery (`pending_prompts` + `remaining_s`) re-shows them when suppression lifts; agents simply get TimedOut if the user never sees the prompt in time.

**Tech:** rusqlite (bundled), tauri tray-icon + menu, core-graphics CGWindowList heuristic for fullscreen detection, Codex CLI (`codex exec`) for the icon asset.

### Task 4: Suppression engine — pause + fullscreen detection

**Files:** Create `src-tauri/src/suppress.rs` (real impl); modify `lib.rs` (notify path), `tray.rs` (wire handlers, dynamic "Pause" title showing remaining time is OPTIONAL — skip unless trivial).

- [ ] `SuppressionState { pause_until: Option<DateTime<Utc>>, hide_in_fullscreen: bool }` in `Arc<Mutex<...>>`, persisted via settings table (survives restart; expired pause ignored on load). `should_suppress() -> bool`: paused OR (hide_in_fullscreen AND frontmost_app_is_fullscreen()).
- [ ] Fullscreen detection (macOS): CGWindowList heuristic — frontmost app's on-screen window at layer 0 whose bounds equal a display's full bounds while the menu bar is hidden on that display; practical shortcut used by menu-bar apps: check `CGWindowListCopyWindowInfo(kCGWindowListOptionOnScreenOnly)` for a frontmost-owning window with bounds == screen frame. Implement in `suppress.rs` behind `#[cfg(target_os = "macos")]` via the `core-graphics` crate; non-macOS: always false. KEEP IT CHEAP: called once per incoming prompt, not polled.
- [ ] Notify path in lib.rs: if `should_suppress()`, skip BOTH emit and show — the prompt stays pending; log one stderr line. On unpause ("Resume now", pause expiry check at next prompt arrival, or fullscreen toggle off): if any `registry.pending()` non-empty → emit newest + show panel (reuse the existing PromptEvent path; remaining_s already handles partial elapse).
- [ ] Pause expiry: lazy (checked in should_suppress) + a tokio timer armed at pause-set that triggers the unpause-replay at expiry.
- [ ] Tests: suppression logic with injected clock/fullscreen fn (trait or fn pointer for testability); replay-on-resume integration-style test at the registry/notify seam.
- [ ] Live verification: pause 15 min via tray → fire demo prompt → nothing shows (screencapture proves) → "Resume now" → prompt appears (if still within timeout). ## Self-review notes
- Recording in the TOOL path (not registry) keeps registry pure and captures elapsed/via from the actual response; CLI asks recorded identically since they go through the same socket tool.
- Suppression gates display only — agents keep the existing TimedOut contract; no protocol change.
- The cold-start replay machinery (pending_prompts/remaining_s, generation guards) is reused for resume-replay — same code path, already race-reviewed.
- Codex is constrained to the icon asset (creative, isolated, fits "created using codex" verbatim) rather than tray code, keeping rmcp/tauri code under our reviewed pipeline.
